### ðŸŽ“ Lessons Learned from Iterative Generation

**What Worked:**

1. **Aspect Ratio Specification Critical**
   - NotebookLM variants were landscape (2752 x 1536)
   - Initial Gemini variants were portrait (1792 x 2400) - FAILED
   - Explicit "LANDSCAPE (16:9)" in prompt fixed it
   - **Lesson**: Always specify aspect ratio explicitly in prompts

2. **Strategic Design Variations**
   - Generated 3 different design approaches: Standard, Card-based, Minimalist
   - Maximized probability of hitting 90%+ threshold
   - V11 (minimalist) achieved 91% vs V9/V10 at 85%
   - **Lesson**: Generate multiple design philosophies, not just minor tweaks

3. **Exact Color Specifications**
   - NotebookLM variants approximated EventAI colors (scored 8/10)
   - Gemini V11 used exact hex codes (scored 10/10)
   - Explicit hex codes in prompt made the difference
   - **Lesson**: Always include exact hex codes, not just color names

4. **White Space as Differentiator**
   - V11's 40%+ white space vs competitors' 30-35%
   - Single biggest factor in V11's superior score
   - Minimalist approach valued over feature-packed designs
   - **Lesson**: "Less is more" - generous white space wins

5. **Gemini Chat > Gemini API**
   - API has free tier quota limits (limit: 0 in December 2025)
   - gemini.google.com has separate quota system
   - Playwright automation successful for chat interface
   - **Lesson**: When API blocked, browser automation viable alternative
